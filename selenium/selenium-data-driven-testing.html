Introduction to Data-Driven Testing

Data-Driven Testing (DDT) separates test logic from test data, allowing the same test to run with multiple data sets. This approach improves test coverage, reduces code duplication, and makes tests easier to maintain.

Benefits of Data-Driven Testing

  • Test same functionality with multiple data sets
  • Separate test logic from test data
  • Easier to maintain and update test data
  • Non-technical users can update test data
  • Improved test coverage with minimal code
  • Reusable test scripts

Excel Utility Class
import org.apache.poi.ss.usermodel.*;
import org.apache.poi.xssf.usermodel.XSSFWorkbook;
import java.io.FileInputStream;

public class ExcelUtils {
    private Workbook workbook;
    private Sheet sheet;
    
    public ExcelUtils(String filePath, String sheetName) {
        try {
            FileInputStream fis = new FileInputStream(filePath);
            workbook = new XSSFWorkbook(fis);
            sheet = workbook.getSheet(sheetName);
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
    
    public int getRowCount() {
        return sheet.getLastRowNum() + 1;
    }
    
    public int getColumnCount() {
        return sheet.getRow(0).getLastCellNum();
    }
    
    public String getCellData(int rowNum, int colNum) {
        Cell cell = sheet.getRow(rowNum).getCell(colNum);
        return getCellValue(cell);
    }
    
    private String getCellValue(Cell cell) {
        if (cell == null) return "";
        
        switch (cell.getCellType()) {
            case STRING:
                return cell.getStringCellValue();
            case NUMERIC:
                return String.valueOf((int) cell.getNumericCellValue());
            case BOOLEAN:
                return String.valueOf(cell.getBooleanCellValue());
            default:
                return "";
        }
    }
    
    public void closeWorkbook() {
        try {
            workbook.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
Excel DataProvider
import org.testng.annotations.DataProvider;

public class TestDataProvider {
    
    @DataProvider(name = "loginData")
    public Object[][] getLoginData() {
        String filePath = "./testdata/TestData.xlsx";
        String sheetName = "LoginData";
        
        ExcelUtils excel = new ExcelUtils(filePath, sheetName);
        
        int rows = excel.getRowCount() - 1; // Exclude header
        int cols = excel.getColumnCount();
        
        Object[][] data = new Object[rows][cols];
        
        for (int i = 0; i < rows; i++) {
            for (int j = 0; j < cols; j++) {
                data[i][j] = excel.getCellData(i + 1, j);
            }
        }
        
        excel.closeWorkbook();
        return data;
    }
}

Write Data to Excel
import org.apache.poi.xssf.usermodel.XSSFWorkbook;
import java.io.FileOutputStream;

public void writeTestResults(String filePath, int row, String result) {
    try {
        FileInputStream fis = new FileInputStream(filePath);
        Workbook workbook = new XSSFWorkbook(fis);
        Sheet sheet = workbook.getSheetAt(0);
        
        Row rowObj = sheet.getRow(row);
        Cell cell = rowObj.createCell(3); // Result column
        cell.setCellValue(result);
        
        fis.close();
        
        FileOutputStream fos = new FileOutputStream(filePath);
        workbook.write(fos);
        fos.close();
        workbook.close();
    } catch (Exception e) {
        e.printStackTrace();
    }
}

CSV Utility Class
import com.opencsv.CSVReader;
import java.io.FileReader;
import java.util.List;

public class CSVUtils {
    
    public static Object[][] readCSV(String filePath) {
        Object[][] data = null;
        
        try {
            CSVReader reader = new CSVReader(new FileReader(filePath));
            List<String[]> allRows = reader.readAll();
            reader.close();
            
            // Skip header row
            int rows = allRows.size() - 1;
            int cols = allRows.get(0).length;
            
            data = new Object[rows][cols];
            
            for (int i = 0; i < rows; i++) {
                data[i] = allRows.get(i + 1);
            }
        } catch (Exception e) {
            e.printStackTrace();
        }
        
        return data;
    }
}

Read CSV Without Library
import java.io.BufferedReader;
import java.io.FileReader;
import java.util.ArrayList;
import java.util.List;

public static Object[][] readCSVManually(String filePath) {
    List<String[]> data = new ArrayList<>();
    
    try (BufferedReader br = new BufferedReader(new FileReader(filePath))) {
        String line;
        boolean isHeader = true;
        
        while ((line = br.readLine()) != null) {
            if (isHeader) {
                isHeader = false;
                continue; // Skip header
            }
            String[] values = line.split(",");
            data.add(values);
        }
    } catch (Exception e) {
        e.printStackTrace();
    }
    
    return data.toArray(new Object[0][]);
}

CSV vs Excel

  • CSV Pros: Simple, lightweight, version control friendly, faster to read
  • CSV Cons: No formatting, single sheet, limited data types
  • Excel Pros: Multiple sheets, formatting, formulas, data validation
  • Excel Cons: Larger files, requires external library

JSON Utility with Gson
import com.google.gson.Gson;
import com.google.gson.JsonObject;
import com.google.gson.JsonArray;
import java.io.FileReader;

public class JSONUtils {
    
    public static Object[][] readJSON(String filePath) {
        Object[][] data = null;
        
        try {
            Gson gson = new Gson();
            FileReader reader = new FileReader(filePath);
            
            JsonObject jsonObject = gson.fromJson(reader, JsonObject.class);
            JsonArray testData = jsonObject.getAsJsonArray("testData");
            
            data = new Object[testData.size()][1];
            
            for (int i = 0; i < testData.size(); i++) {
                LoginData loginData = gson.fromJson(testData.get(i), LoginData.class);
                data[i][0] = loginData;
            }
            
            reader.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
        
        return data;
    }
}

Database Utility Class
import java.sql.*;
import java.util.ArrayList;
import java.util.List;

public class DatabaseUtils {
    private static Connection connection;

    private static final String URL = "jdbc:mysql://localhost:3306/testdb";
    private static final String USER = "root";
    private static final String PASSWORD = "password";

    public static Connection getConnection() {
        try {
            if (connection == null || connection.isClosed()) {
                connection = DriverManager.getConnection(URL, USER, PASSWORD);
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
        return connection;
    }

    public static void closeConnection() {
        try {
            if (connection != null && !connection.isClosed()) {
                connection.close();
            }
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}
Database DataProvider
@DataProvider(name = "dbData")
public Object[][] getDBData() {
    List<Object[]> data = new ArrayList<>();

    try {
        Connection conn = DatabaseUtils.getConnection();
        String query = "SELECT username, password, expected_result FROM login_test_data";

        Statement stmt = conn.createStatement();
        ResultSet rs = stmt.executeQuery(query);

        while (rs.next()) {
            String username = rs.getString("username");
            String password = rs.getString("password");
            String expectedResult = rs.getString("expected_result");

            data.add(new Object[]{username, password, expectedResult});
        }

        rs.close();
        stmt.close();
    } catch (SQLException e) {
        e.printStackTrace();
    }

    return data.toArray(new Object[0][]);
}

Parameterized Query
public Object[][] getDataByEnvironment(String environment) {
    List<Object[]> data = new ArrayList<>();

    try {
        Connection conn = DatabaseUtils.getConnection();
        String query = "SELECT * FROM test_data WHERE environment = ?";

        PreparedStatement pstmt = conn.prepareStatement(query);
        pstmt.setString(1, environment);

        ResultSet rs = pstmt.executeQuery();

        while (rs.next()) {
            data.add(new Object[]{
                rs.getString("username"),
                rs.getString("password"),
                rs.getString("expected_result")
            });
        }

        rs.close();
        pstmt.close();
    } catch (SQLException e) {
        e.printStackTrace();
    }

    return data.toArray(new Object[0][]);
}

Database Best Practices

  • Use connection pooling for better performance
  • Always close connections in finally block or use try-with-resources
  • Use PreparedStatement to prevent SQL injection
  • Store credentials in config files, not in code
  • Use separate test database, not production

Properties Reader
import java.io.FileInputStream;
import java.util.Properties;

public class ConfigReader {
    private static Properties properties;

    static {
        try {
            FileInputStream fis = new FileInputStream("./config/config.properties");
            properties = new Properties();
            properties.load(fis);
            fis.close();
        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    public static String get(String key) {
        return properties.getProperty(key);
    }

    public static String getBaseUrl() {
        return get("base.url");
    }

    public static String getBrowser() {
        return get("browser");
    }

    public static int getTimeout() {
        return Integer.parseInt(get("timeout"));
    }
}
Using Properties in Tests

TestNG Parameters

TestNG allows passing parameters from testng.xml directly to test methods.

testng.xml with Parameters
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE suite SYSTEM "https://testng.org/testng-1.0.dtd">
<suite name="Data-Driven Suite">

    <!-- Suite-level parameters -->
    <parameter name="browser" value="chrome"/>
    <parameter name="baseUrl" value="https://example.com"/>

    <test name="Login Tests">
        <!-- Test-level parameters -->
        <parameter name="username" value="testuser"/>
        <parameter name="password" value="Test@123"/>

        <classes>
            <class name="tests.LoginTest"/>
        </classes>
    </test>

</suite>
Using Parameters in Test
import org.testng.annotations.Parameters;

public class LoginTest {

    @Parameters({"browser", "baseUrl"})
    @BeforeClass
    public void setup(String browser, String baseUrl) {
        if (browser.equals("chrome")) {
            driver = new ChromeDriver();
        } else if (browser.equals("firefox")) {
            driver = new FirefoxDriver();
        }
        driver.get(baseUrl);
    }

    @Parameters({"username", "password"})
    @Test
    public void testLogin(String username, String password) {
        driver.findElement(By.id("username")).sendKeys(username);
        driver.findElement(By.id("password")).sendKeys(password);
        driver.findElement(By.id("loginBtn")).click();
    }
}
Optional Parameters
import org.testng.annotations.Optional;

Best Practices

Data Management

  • Keep test data separate from test code
  • Use meaningful data names and comments
  • Version control your test data files
  • Create data for positive and negative scenarios

File Organization

  • Use a dedicated testdata folder
  • Separate data by feature or module
  • Use descriptive file names
  • Create environment-specific data files

Security

  • Never commit sensitive data to VCS
  • Use environment variables for credentials
  • Encrypt sensitive test data
  • Use .gitignore for config files

Performance

  • Cache data for repeated reads
  • Close file/DB connections properly
  • Use lazy loading for large datasets
  • Parallel execution for independent data

Choosing the Right Data Source
Data Source    | Best For                      | Pros                    | Cons
---------------|-------------------------------|-------------------------|------------------
Excel          | Business users, complex data  | Familiar, multi-sheet   | Library required
CSV            | Simple data, CI/CD            | Lightweight, VCS-friendly| No formatting
JSON           | API data, nested structures   | Flexible, readable      | Complex parsing
Database       | Dynamic data, large datasets  | Real-time, scalable     | Setup overhead
Properties     | Config, simple key-value      | Built-in Java support   | Limited structure
TestNG XML     | Static parameters             | No code change needed   | Not scalable

Key Takeaways

  • Choose data source based on data complexity and team skills
  • Excel works best when business users need to maintain test data
  • CSV is ideal for simple data and CI/CD pipelines
  • JSON excels at API testing and complex nested data
  • Database is best for large, dynamic, or shared datasets
  • Properties files work well for configuration and environment settings
