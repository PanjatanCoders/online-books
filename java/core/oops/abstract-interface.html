1. Introduction to Abstraction

What is Abstraction?

Abstraction hides implementation details while exposing only essential features.

Basic Example

// Abstract class
public abstract class Animal {
    protected String name;
    
    public Animal(String name) {
        this.name = name;
    }
    
    // Concrete method
    public void sleep() {
        System.out.println(name + " is sleeping");
    }
    
    // Abstract method
    public abstract void makeSound();
}

class Dog extends Animal {
    public Dog(String name) {
        super(name);
    }
    
    @Override
    public void makeSound() {
        System.out.println(name + " barks!");
    }
}

Benefits

  • Reduces complexity
  • Increases code reusability
  • Supports polymorphism
  • Enhances maintainability

Types of Abstraction

  • Abstract Classes (0-100%)
  • Interfaces (100%)
  • Concrete Classes (0%)

2. Abstract Classes

Key Characteristics

  • Cannot be instantiated directly
  • Can contain both abstract and concrete methods
  • Can have constructors and instance variables
  • Subclasses must implement abstract methods

Shape Example

public abstract class Shape {
    protected String color;
    
    public Shape(String color) {
        this.color = color;
    }
    
    // Concrete method
    public String getColor() {
        return color;
    }
    
    // Abstract methods
    public abstract double calculateArea();
    public abstract void draw();
}

class Circle extends Shape {
    private double radius;
    
    public Circle(String color, double radius) {
        super(color);
        this.radius = radius;
    }
    
    @Override
    public double calculateArea() {
        return Math.PI * radius * radius;
    }
    
    @Override
    public void draw() {
        System.out.println("Drawing " + color + " circle");
    }
}
Remember: Abstract methods cannot be private, static, or final.

3. Interfaces

Interface Characteristics

  • All methods are public and abstract by default
  • All variables are public, static, and final
  • Can have default and static methods (Java 8+)
  • Supports multiple inheritance

Payment Interface Example

public interface PaymentProcessor {
    // Constants
    double MIN_AMOUNT = 0.01;
    
    // Abstract methods
    boolean processPayment(double amount);
    boolean refund(String transactionId);
    
    // Default method (Java 8+)
    default boolean validateAmount(double amount) {
        return amount >= MIN_AMOUNT;
    }
    
    // Static method (Java 8+)
    static String formatCurrency(double amount) {
        return String.format("$%.2f", amount);
    }
}

class CreditCardProcessor implements PaymentProcessor {
    @Override
    public boolean processPayment(double amount) {
        if (!validateAmount(amount)) {
            return false;
        }
        System.out.println("Processing credit card payment: " + 
                         PaymentProcessor.formatCurrency(amount));
        return true;
    }
    
    @Override
    public boolean refund(String transactionId) {
        System.out.println("Refunding: " + transactionId);
        return true;
    }
}
Tip: Use interfaces for contracts and capabilities (CAN-DO relationships).

4. Abstract Classes vs Interfaces

Aspect Abstract Class Interface
Keyword abstract class interface
Methods Abstract + Concrete Abstract + Default/Static
Variables Instance variables Constants only
Constructor Yes No
Inheritance Single (extends) Multiple (implements)

Use Abstract Classes When:

  • Sharing common code
  • IS-A relationship
  • Need constructors
  • Protected/private members

Use Interfaces When:

  • Defining contracts
  • CAN-DO relationship
  • Multiple inheritance needed
  • Maximum flexibility

5. Best Practices

Interface Design

  • Keep interfaces small and focused
  • Use meaningful names
  • Provide default implementations when helpful
  • Document contracts clearly

Abstract Classes

  • Use for template method pattern
  • Provide common implementation
  • Use protected for subclass access
  • Consider final methods for invariants

Good Practice Example

// Good: Segregated interfaces
interface Worker {
    void work();
}

interface Programmer extends Worker {
    void code();
    
    default void debug() {
        System.out.println("Debugging...");
    }
}

interface Designer extends Worker {
    void design();
}

// Implementation can choose specific capabilities
class FullStackDeveloper implements Programmer, Designer {
    @Override
    public void work() {
        code();
        design();
    }
    
    @Override
    public void code() {
        System.out.println("Writing code");
    }
    
    @Override
    public void design() {
        System.out.println("Creating designs");
    }
}
Avoid: Fat interfaces with too many methods (violates Interface Segregation Principle).

6. Summary - What We've Learned

Key Takeaways

Throughout this guide, we've explored the fundamental concepts of abstraction in Java through abstract classes and interfaces.

Abstract Classes

  • Provide partial abstraction (0-100%)
  • Can contain both abstract and concrete methods
  • Support constructors and instance variables
  • Enable IS-A relationships with shared code
  • Perfect for template method pattern

Interfaces

  • Provide complete abstraction (100%)
  • Support multiple inheritance
  • Define contracts and capabilities
  • Enable CAN-DO relationships
  • Support default and static methods (Java 8+)

Design Principles

  • Follow SOLID principles
  • Keep interfaces small and focused
  • Use meaningful names and documentation
  • Choose based on relationship type
  • Prioritize composition over inheritance

7. Common Interview Questions

Test your understanding with these frequently asked interview questions about abstract classes and interfaces.

Conceptual Questions

Q1: What is the difference between abstract class and interface?
Hint: Think about inheritance, method types, and variables
Q2: Can we instantiate an abstract class? Why or why not?
Hint: Consider the purpose of abstraction
Q3: What are default methods in interfaces? When were they introduced?
Hint: Java 8 feature for interface evolution
Q4: Can an interface extend another interface? Can it extend multiple interfaces?
Hint: Interface inheritance vs class inheritance

Coding Questions

Q5: Write an abstract class Vehicle with concrete and abstract methods. Create two subclasses Car and Bike.
Hint: Include common properties and vehicle-specific behaviors
Q6: Create an interface Drawable with a draw() method. Implement it in Circle and Rectangle classes.
Hint: Use method overriding in implementations
Q7: Demonstrate multiple interface implementation with a class that implements both Flyable and Swimmable interfaces.
Hint: Think of animals that can do both

Advanced Questions

Q8: What happens if a class implements two interfaces with the same default method signature?
Hint: Diamond problem resolution
Q9: Can we have private methods in interfaces? Which Java version introduced this?
Hint: Java 9 feature for code reuse in default methods
Q10: Explain the Template Method pattern using abstract classes.
Hint: Define algorithm structure, let subclasses implement steps
Q11: When would you choose composition over inheritance while working with abstractions?
Hint: Favor composition for flexibility and loose coupling

Interview Tips

Be Clear: Explain concepts with real-world examples
