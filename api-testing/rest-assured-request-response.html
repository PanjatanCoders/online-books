Request & Response Handling

Deep dive into HTTP methods, headers, parameters, and advanced response validation with JsonPath.

HTTP Methods

PUT - Full Update

PUT replaces the entire resource with the provided data:

@Test
public void updateUserWithPut() {
    Map<String, String> updatedUser = new HashMap<>();
    updatedUser.put("name", "Updated Name");
    updatedUser.put("job", "Senior Engineer");

    given()
        .baseUri("https://reqres.in")
        .contentType(ContentType.JSON)
        .body(updatedUser)
    .when()
        .put("/api/users/2")
    .then()
        .statusCode(200)
        .body("name", equalTo("Updated Name"))
        .body("job", equalTo("Senior Engineer"))
        .body("updatedAt", notNullValue());
}

PATCH - Partial Update

PATCH updates only the specified fields:

@Test
public void partialUpdateWithPatch() {
    // Only updating the job field
    String patchBody = """
        {
            "job": "Tech Lead"
        }
        """;

    given()
        .baseUri("https://reqres.in")
        .contentType(ContentType.JSON)
        .body(patchBody)
    .when()
        .patch("/api/users/2")
    .then()
        .statusCode(200)
        .body("job", equalTo("Tech Lead"))
        .body("updatedAt", notNullValue());
}

PUT vs PATCH

PUT PATCH
Replaces entire resource Updates specified fields only
Must send all fields Send only changed fields
Idempotent Not necessarily idempotent

Validating Response Headers

@Test
public void validateResponseHeaders() {
    given()
        .baseUri("https://reqres.in")
    .when()
        .get("/api/users")
    .then()
        .statusCode(200)
        // Exact match
        .header("Content-Type", "application/json; charset=utf-8")

        // Partial match
        .header("Content-Type", containsString("application/json"))

        // Check header exists
        .header("Date", notNullValue())

        // Multiple header validations
        .headers("Content-Type", containsString("json"),
                 "Transfer-Encoding", equalTo("chunked"));
}

@Test

JsonPath Expressions

JsonPath is a query language for JSON, similar to XPath for XML.

Sample Response

{
    "page": 1,
    "total": 12,
    "data": [
        {
            "id": 1,
            "email": "george@reqres.in",
            "first_name": "George",
            "last_name": "Bluth",
            "avatar": "https://reqres.in/img/faces/1-image.jpg"
        },
        {
            "id": 2,
            "email": "janet@reqres.in",
            "first_name": "Janet",
            "last_name": "Weaver",
            "avatar": "https://reqres.in/img/faces/2-image.jpg"
        }
    ],
    "support": {
        "url": "https://reqres.in",
        "text": "Support message"
    }
}

JsonPath Syntax

Expression Description Result
page Root level field 1
data[0].email First array element's email "george@reqres.in"
data[-1].id Last array element's id 2
data.id All ids from array [1, 2]
data.size() Array size 2
support.url Nested object field "https://reqres.in"

JsonPath in Validations

@Test
public void jsonPathValidations() {
    given()
        .baseUri("https://reqres.in")
    .when()
        .get("/api/users?page=1")
    .then()
        .statusCode(200)

        // Simple field
        .body("page", equalTo(1))

        // Array element by index
        .body("data[0].id", equalTo(1))
        .body("data[0].email", equalTo("george.bluth@reqres.in"))

        // Array operations
        .body("data.size()", equalTo(6))
        .body("data.id", hasItems(1, 2, 3))

        // Nested objects
        .body("support.url", containsString("reqres"))

        // Collection of values
        .body("data.email", everyItem(containsString("@")));
}

Groovy GPath Expressions

Rest Assured uses Groovy's GPath for advanced queries:

@Test
public void gPathExpressions() {
    given()
        .baseUri("https://reqres.in")
    .when()
        .get("/api/users?page=1")
    .then()
        .statusCode(200)

        // Find single item matching condition
        .body("data.find { it.id == 1 }.email", equalTo("george.bluth@reqres.in"))

        // Find all items matching condition
        .body("data.findAll { it.id > 3 }.size()", greaterThan(0))

        // Collect specific field from all items
        .body("data.collect { it.first_name }", hasItem("George"))

        // Sum, min, max
        .body("data.id.sum()", greaterThan(0))
        .body("data.id.min()", equalTo(1))
        .body("data.id.max()", equalTo(6));
}

Extracting with JsonPath

@Test
public void extractWithJsonPath() {
    Response response = given()
        .baseUri("https://reqres.in")
    .when()
        .get("/api/users?page=1")
    .then()
        .extract().response();

    // Using JsonPath object
    JsonPath jsonPath = response.jsonPath();

    // Extract single value
    int page = jsonPath.getInt("page");
    String firstEmail = jsonPath.getString("data[0].email");

    // Extract list
    List<String> emails = jsonPath.getList("data.email");
    List<Integer> ids = jsonPath.getList("data.id", Integer.class);

    // Extract as Map
    Map<String, Object> firstUser = jsonPath.getMap("data[0]");

    // Extract list of objects
    List<Map<String, Object>> allUsers = jsonPath.getList("data");

    System.out.println("Emails: " + emails);
    System.out.println("First user: " + firstUser);
}

JSON Schema Validation

Validate that API responses conform to a defined schema structure.

Create JSON Schema

Save as src/test/resources/schemas/user-schema.json:

{
    "$schema": "http://json-schema.org/draft-07/schema#",
    "type": "object",
    "required": ["data", "support"],
    "properties": {
        "data": {
            "type": "object",
            "required": ["id", "email", "first_name", "last_name"],
            "properties": {
                "id": { "type": "integer" },
                "email": { "type": "string", "format": "email" },
                "first_name": { "type": "string" },
                "last_name": { "type": "string" },
                "avatar": { "type": "string", "format": "uri" }
            }
        },
        "support": {
            "type": "object",
            "properties": {
                "url": { "type": "string" },
                "text": { "type": "string" }
            }
        }
    }
}

Validate Against Schema

import static io.restassured.module.jsv.JsonSchemaValidator.*;

@Test
public void validateJsonSchema() {
    given()
        .baseUri("https://reqres.in")
    .when()
        .get("/api/users/2")
    .then()
        .statusCode(200)
        .body(matchesJsonSchemaInClasspath("schemas/user-schema.json"));
}

@Test
public void validateSchemaFromFile() {
    File schemaFile = new File("src/test/resources/schemas/user-schema.json");

    given()
        .baseUri("https://reqres.in")
    .when()
        .get("/api/users/2")
    .then()
        .statusCode(200)
        .body(matchesJsonSchema(schemaFile));
}

Schema Validation Benefits

  • Catches breaking API changes early
  • Validates data types automatically
  • Ensures required fields are present
  • Documents API contract
